Getting Started

Installing Node

Node.js can be downloaded here. After downloading run the installer. Open a command prompt and run 

$ node --version
to see if the correct version is installed.

Installing npm packages

The Node.js install includes a dependence management tool called npm. npm installs dependence from the npm repo. Dependence for a project can be defined in packages. Simply running 

$ npm install
in the project's root will install all of the dependences including Grunt.

Installing Sass

Installing Jekyll

Jekyll is a static site generator that powers this documentation. Its build on ruby and can be installed with RubyGems. Once you have ruby installed and can access gem binaries from the command line it should be easy to install and run.

  1. Install Jekyll. 
    $ gem install jekyll
  2. Install Rouge for syntax highlighting. 
    $ gem install rouge
  3. In the project's root run 
    $ jekyll serve --config _config.yml
  4. Now you can navigate to http://localhost:9001/ui/ in the browser.

Committing Guidelines

Pre-commit

Before you commit...

  • Check that you have the correct branch.
  • Check that you have the latest branch.
  • Run grunt lint:all to check for syntax errors.

Commit Messages

Commit messages should be contain:

  • Jira issue number
  • Short description of the Jira issue. Usually the Jira issue title.
  • Quick description of the change. Try to expand upon what can be understood from the change.

Optionally, a more detailed description of the change can be added after adding a new line break.

Example: 

    DTC-1234 - Inseams not selected correctly - Fixed jQuery selectors

      - Added ':first' selector to the sku sort.
      - Added 'input[type=text]' to the product form.
